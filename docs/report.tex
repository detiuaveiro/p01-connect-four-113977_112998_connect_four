    \subsection{Bitboards}
    This represents the board using two large integers, one for each player. Each bit in the integer corresponds to a square on the grid ($index = row \times 7 + column$)

    \begin{center}
        Player 1: \texttt{171832246272}\\
        Player 2: \texttt{554051043328}
    \end{center}

    If Player 1 has a piece at Row 5, Col 0, you set the $35^{th}$ bit ($2^{35}$). By summing these powers of 2 (using bitwise OR), you get a single number that perfectly describes all of one player's pieces. This is the fastest way to check for wins using bitwise masks.

    171832246272 representa em decimal o número binário:\\
    \texttt{0000000000000000000000001010000000001000000000000000000000000000}

    $7 \times 6 = 42$ squares = 42 bits (usamos 64 bits, sobrando 22 bits para outras informações ou para evitar overflow)

    \subsection{Zobrist Hashing}
    This is the standard for Transposition Tables. It converts the entire board into a single unique 64-bit number using the XOR operator.

    \begin{center}
        Zobrist Hash: \texttt{11436696887795538186}\\
        (Example based on a seeded random table)
    \end{center}

O melhor para verificar vitórias é o Bitboards

O melhor para guardar é o Zobrist Hashing
